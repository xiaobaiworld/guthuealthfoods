# Guthu Health Foods

## 项目定位

Guthu Health Foods 是一个以健康食品、营养知识和双语内容为核心的静态内容网站。

项目目标不是只做一个演示首页，而是逐步建设成一个可持续扩充的内容型站点，最终承载 100+ 页面规模的公开内容，并兼顾：

- 人类用户的清晰浏览体验
- 搜索引擎可索引性
- LLM 易解析、易引用、易检索
- 纯静态导出与低运维复杂度

## 当前技术方向

- Next.js App Router
- React 19
- 静态导出：`output: 'export'`
- 双语路由：`/en` 与 `/zh`
- 内容来源：计划使用 Markdown / JSON 驱动

## 内容规模目标

当前规划不应停留在少量演示页。

建议按页面家族推进到 100+ 页面规模：

1. 双语首页：2 页
2. 健康食品详情页：40 到 60 个主题，每个主题中英双语，共 80 到 120 页
3. 列表 / 分类聚合页：8 到 12 个主题，每个主题中英双语，共 16 到 24 页
4. 品牌 / 方法 / 法律说明页：3 到 5 类，每类中英双语，共 6 到 10 页

这意味着项目合理的公开页面规模会落在：

- 104 到 156 页

因此，项目架构必须从一开始就按“可批量生产、可批量校验、可批量发布”的模式设计。

## 开发原则

1. 先建内容模型，再建页面模板，再批量扩展内容。
2. 先保证真实可索引的 URL 和 metadata，再接入 Google 分析。
3. 每次只推进一个明确子任务，并做本地 commit。
4. 在达到一批较完整成果之前，不急于推送远程服务器。
5. 除 Google 账户、域名、DNS、GA4 等必须人工提供的信息外，其他步骤尽量由本地自动化完成。

## 推荐执行顺序

### 第一段：底层能力

- 建立内容解析器
- 设计统一 frontmatter / JSON 数据模型
- 建立食品详情页、列表页、静态基础页的页面模板

### 第二段：首批可用发布集

- 完成首批真实内容样本
- 跑通双语详情页和列表页
- 补齐 metadata、JSON-LD、sitemap、robots
- 完成第一批本地 QA

### 第三段：规模化扩展

- 建立 40 到 60 个食品主题 inventory
- 按批次生成和发布页面
- 每批完成后做本地 commit，不立即推远程
- 当站点达到较完整规模后，再统一部署到远程服务器

### 第四段：上线后接入

- 连接 Google Search Console
- 接入 GA4
- 验证 sitemap、收录、基础事件
