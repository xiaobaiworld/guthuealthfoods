# Guthu Health Foods - 智能体开发指南

## 1. 项目概览
项目名称：Guthu Health Foods
描述：一个极具现代感、定位高端且以健康及绿色为主题的数据型网站。
目标受众：人类用户与 LLM（大语言模型）爬虫/信息抓取器。
多语言支持：英文与中文（双语）。

## 2. 核心要求
- **SEO 与大语言模型（LLM）友好**：
  - 必须严格维护和优化 Meta 标签、OpenGraph 以及结构化数据（JSON-LD）以确保最佳的 SEO 效果。
  - 必须采用语义化的 HTML 结构（例如：`<article>`、`<section>`、`<nav>`），确保大语言模型能够高效地解析网页结构和内容。
  - 内容（结构）必须与样式（CSS）实现清晰的分离。
- **设计与审美**：
  - 网站视觉设计**必须**令人惊叹。务必采用现代化的 UI/UX 趋势（如适用可采用毛玻璃效果、平滑渐变、细腻的微交互动画）。
  - 色板：必须契合“健康”与“绿色”的主题（例如：自然灵动的绿色系、干净极致的白色系、沉稳质朴的大地色系）。坚决避免使用随处可见的通用普通色彩。
  - 排版与字体：应用现代且易读性强的字体（建议使用如 Inter、Roboto 等）。
- **国际化（i18n）**：
  - 所有的内容数据与架构必须无缝支撑至少中英（ZH/EN）双语切换。
- **可扩展性**：
  - 架构设计必须能够轻松应对未来更多数据体系和材料的扩充。

## 3. 技术栈规则
- **框架**：强烈推荐使用 Next.js 进行静态站点生成（SSG），因为它完美契合 SEO 诉求以及严格的数据结构化需求。样式方面搭配 Vanilla CSS（除非用户明确要求使用 Tailwind）。
- **代码仓库管理**：使用 Git 进行版本控制。每次提交（Commit）必须保持原子性，且需具备清晰的备注说明。
- **开发流程**：永远谋定而后动。在执行任何代码前都要规划好，并持续同步更新 `agent.md` 或 `task.md` 以反映需求的变化。

## 4. 后续智能体工作流
在实现新特性或增添新数据资料时，必须遵循以下步骤：
1. 参照设计系统（Design System）进行验证，确保符合高端视觉的基调。
2. 验证所有新增的路由及页面都包含了正确的 SEO Meta 标签。
3. 确保所有文本内容已被正确提取，或在结构上已准备好进行中英双语的本地化适配。
4. 结构化输出极其干净整洁的 JSON-LD 或 Markdown 等效数据格式，以便大语言模型（LLM）直接摄取和阅读。
